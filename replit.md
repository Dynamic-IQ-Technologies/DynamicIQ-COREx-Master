## Overview

Dynamic.IQ.MRP is a comprehensive Manufacturing Resource Planning (MRP) system built with Flask, designed to optimize production processes, inventory, Bill of Materials (BOM), work orders, and purchase orders. It offers robust role-based access control, efficient material tracking, production planning, supplier management, and report generation. The system aims to provide a scalable, user-friendly solution to enhance operational efficiency and deliver critical business insights.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### UI/UX Decisions

The system features a responsive and modern user interface built with Bootstrap 5 and Bootstrap Icons, ensuring a consistent layout via a base template. It utilizes card-based and accordion-style displays, including an executive financial accounting dashboard with interactive Chart.js visualizations and color-coded KPI indicators.

### Technical Implementations

The backend is developed with Flask using Blueprints and a SQLite database (`mrp.db`). It features session-based authentication, comprehensive role-based access control (Admin, Planner, Production Staff, Procurement, Accountant), and an `MRPEngine` for core MRP logic like recursive BOM explosion and automatic sequential numbering. An Audit Trail System logs all CUD operations. The frontend uses Jinja2 for templating.

**Key Modules and Features:**
-   **Products & BOM**: Manages product data, multi-level BOMs with revision control, and interactive views. **Product Cost Management**: Product costs are NOT manually editable on the product record; instead, costs are automatically updated during the receiving process based on actual purchase prices from PO lines. New products start with $0.00 cost until first receipt. **Product-Level UOM Conversion Management with Versioning**: Each product page includes a comprehensive unit conversion management system with full version control. Features include **full CRUD operations** (Create, Read, Update, Delete) for UOM conversions with dedicated edit functionality, **multiple version support** allowing users to create and track different conversion rates over time with version numbers, effective dates, and detailed version notes, base UOM designation for inventory tracking (always factor 1.0), dynamic conversion preview showing real-time calculations (e.g., "1 BOX = 12 EA"), purchase/issue UOM defaults for auto-selection, validation to prevent deleting base UOM, version history viewer showing all historical conversion rates with activation capability, and comprehensive audit trail logging of all conversion changes and version activations. The edit modal intelligently locks the UOM selector (since UOM cannot be changed after creation), displays context-appropriate titles ("Add UOM Conversion" vs "Edit UOM Conversion"), and provides a "Create New Version" option when editing existing conversions to preserve historical data. Only one version per product-UOM combination is active at a time, ensuring system-wide consistency while maintaining complete version history for compliance and auditing. The system automatically calculates base quantities for accurate inventory tracking while displaying both order and base units throughout Purchase, Sales, and Inventory modules.
-   **Inventory**: Tracks stock levels, supports serialized products with unique serial number tracking, and allows manual adjustments. **Cost Visibility**: Displays Unit Cost and Total Inventory Value columns for each item with real-time calculation and a grand total inventory value summary. Includes NULL cost handling with COALESCE to prevent runtime errors when product costs are missing.
-   **Work Orders**: Manages production orders with disposition types (Manufacture, Repair, Overhaul, Teardown, Inspect), **customer association** (linked to customer database via foreign key with full contact details), status tracking, cost allocation, and integrated task/labor planning. **Operational Status Tracking**: Flexible workflow status field with 14 predefined stages (Receiving, Incoming Inspection, Clean & Disassembly, Preliminary Inspection, Requirements, Pricing, Quoting, Awaiting Customer Approval, Material Processing, Production, Final Inspection, Accounting, Shipping, Stock) allowing detailed tracking of work order progression through manufacturing and service processes. **Full CRUD Operations**: Create, view, edit (with restrictions on completed orders), and status updates with automatic material requirement recalculation when product changes. **WIP → Finished Goods Transfer**: Automatic GL posting when work orders complete, transferring accumulated costs (Material + Labor + Overhead) from WIP (1140) to Finished Goods Inventory (1150), with inventory quantity updates and product cost recalculation. **Work Order Quote System**: Generate, manage, and share professional quotes directly from work orders with automatic data population (including customer name), customizable pricing with configurable markup percentage (applied to parts only), dedicated header fields for Labor/Consumables/Other Fees (not included as line items), PDF generation with company branding, and status workflow (Draft → Submitted → Approved/Rejected → Converted). Features multi-line item support for parts, tax calculation on total amount (Parts + Labor + Consumables + Other Fees), server-side financial calculations for security, and comprehensive audit trails. **Material Allocation and Issuance Workflow**: Advanced material management system with two-phase control for production floor. Features include **Material Allocation** (reserve inventory for work orders without physical movement, tracks allocated_quantity and allocation_status with user/timestamp tracking, validates against available inventory, prevents over-allocation beyond requirements), **Material Issuance** (physical transfer to production floor, deducts from inventory, posts to GL with DR: WIP 1140, CR: Inventory 1100, tracks issued_quantity with user/timestamp), **Material Returns** (reverse issuance process, adds back to inventory, reverses GL posting with DR: Inventory, CR: WIP), **Material Deallocation** (releases reserved but not issued materials), color-coded status badges (Not Allocated=secondary, Partially Allocated=warning, Fully Allocated=primary, Partially Issued=info, Fully Issued=success), comprehensive material requirements table showing Required/Allocated/Issued quantities with available inventory, dropdown action menus per line item with role-based access controls (Admin, Planner, Production Staff), 4 interactive modals with validation and informational alerts, automatic status calculation based on quantities, inventory validation preventing over-issuance, and full audit trail logging for all allocation/issuance actions. Database schema includes allocated_quantity, issued_quantity, allocation_status, allocated_by, allocated_at, issued_by, issued_at columns in material_requirements table.
-   **Purchase Orders**: Supports multi-line procurement, supplier relationships, dynamic line item management, partial/full receiving, and secure edit capabilities with audit trails. **UOM Conversion Management**: Advanced unit conversion system that handles different units of measure (boxes, cartons, pieces, liters, etc.) with automatic conversion to base units for accurate inventory tracking. Features include product-specific conversion factors from `product_uom_conversions` table, automatic calculation and storage of both order quantity (in purchase UOM) and base quantity (in base UOM) on each PO line, **enhanced conversion display in PO view** showing base quantity prominently with intelligent conversion indicator (e.g., "48 EA" with "10 BOX = 48 EA" below when UOMs differ, avoiding redundant display when units match), API endpoints (`/api/product-uom-conversions/{product_id}` and `/api/calculate-conversion`) for dynamic UOM dropdown population and JavaScript-powered conversion calculations, and receiving process integration that converts received quantities to base units before updating inventory. Database schema includes `base_quantity`, `base_uom_id`, and `conversion_factor_used` columns on `purchase_order_lines` table.
-   **Material Management**: Comprehensive systems for Receiving, Issuance, and Returns, all with automatic GL posting. **Receiving Cost Management**: When materials are received, the product cost is automatically updated to reflect the actual purchase cost from the PO line (unit price / conversion factor for base UOM). The receiving interface displays unit cost and dynamically calculates total receiving value as quantity is entered. This ensures inventory valuation is based on actual purchase costs, not product master costs.
-   **Suppliers & UOM**: Manages supplier information and a centralized Unit of Measure master.
-   **Time Tracking**: Employee clock-in/out system with real-time tracking, cost calculation, and a dedicated mobile-friendly Clock Station with PIN authentication and security features.
-   **Sales Module**: Comprehensive sales order management including customer CRUD, various order types (Outright Sales, Exchanges, Managed Repair), advanced line management with workflow, tax calculation, inventory integration (including serialized products), a 5-state order workflow, and robust validation (stock, pricing, discounts, credit limits). Includes Core Due Tracking for exchanges. **Line-Level Inventory Allocation with Serial Number Tracking**: Individual line items track allocated inventory with fields for allocated_quantity, allocation_status (Pending/Allocated/Partially Allocated/Backordered), released_to_shipping_at timestamp, serial_number (for serialized products), and shipped_quantity. Features allocation buttons per line, **automatic serial number population** from serialized inventory during allocation, duplicate serial number validation preventing same serial from being allocated to multiple active orders, visual status indicators with color-coded badges displaying serial numbers as info badges, "Allocate All Lines" bulk operation, and validation preventing release of unallocated items. **Line-Level Release to Shipping**: Each line can be individually released to shipping queue after allocation, supporting partial shipments and mixed-status orders where some lines ship while others remain pending. Role-based controls restrict allocation/release to Admin and Planner roles.
-   **Shipping & Receiving Module**: Manages shipment lifecycle for Sales and Work Orders with multi-line support, tracking information, and status workflows. Includes package details and inventory integration. **Pending Shipments Workflow**: Controlled release process where Sales/Order Management releases orders to a Pending Shipments queue, then Shipping/Production personnel confirm and finalize shipments with inventory deduction. Features shipment stage tracking (Pending → Confirmed), automatic status updates, carrier/tracking information capture, and comprehensive audit logging.
-   **Invoice Management Module**: Comprehensive billing and A/R system generating invoices from Sales/Work Orders, supporting a full invoice lifecycle (Draft → Approved → Posted → Paid), multi-line items, and an Invoice Dashboard with advanced filtering and KPIs. **Automatic Revenue Recognition**: When invoices are Posted, the system automatically creates GL entries (DR: Accounts Receivable 1120, CR: Sales Revenue 4100) ensuring accurate financial reporting.
-   **Service Management Module**: Comprehensive service work order system for both standard Service and NDT (Non-Destructive Testing) work orders. Features include customer association, equipment tracking (description, serial, model), multi-line labor tracking with automatic cost calculation (Regular, Service Rate with custom rates, Overtime 1.5x, NDT 1.3x rates), materials allocation with optional inventory deduction, expense tracking (travel, subcontractors, consumables), status workflow (Open → In Progress → Completed → Invoiced), approval process, and automatic invoice generation. Supports service types: Service, NDT, Repair, Maintenance, Inspection. Priority levels (Low/Medium/High) and assignment to technicians. Cost tracking shows labor, materials, and expenses subtotals with automatic total calculation. **Conditional Service Rate Field**: Labor entry forms include a conditional "Service Rate" field that only appears when "Service Rate" labor type is selected, allowing custom hourly rates per service entry with JavaScript-based field toggling and validation. Database schema includes `service_rate` column in `service_wo_labor` table. **Part Selection During Creation**: Dynamic part/material selection when creating service work orders with product dropdown showing part numbers and descriptions, optional description field with auto-fill from product data, conditional serial number selection for serialized products (fetches available serials from inventory via API), quantity input, and automatic cost calculation using product cost as unit price. Materials table displays serial numbers with info badges for serialized items. Database schema includes `serial_number` column in `service_wo_materials` table. **Invoice Generation Workflow**: Complete service work order → Approve for invoicing (Admin/Planner) → Generate invoice with automatic line creation from labor entries (with employee details and hours), materials (with product codes), and expenses → Automatic invoice numbering (INV-00001, INV-00002, etc.) → Links to generated invoice for viewing and management. Prevents duplicate invoicing and enforces completion/approval requirements.

### System Design Choices

-   **Inventory Management**: Real-time stock level tracking, low stock alerts, and automatic updates.
-   **Reporting System**: Provides various reports including inventory valuation, work order cost analysis, material requirements, material usage, and active labor.
-   **Accounting System**: Features a Chart of Accounts (COA), General Ledger (GL), Manual Journal Entries, and automatic GL posting for inventory transactions. It includes financial reports (Trial Balance, Balance Sheet, Income Statement) and an Accounts Payable (A/P) module with automated vendor invoice creation, payment tracking with GL posting, aging reports, and an Executive Accounting Dashboard providing real-time financial KPIs and interactive visualizations. **GL Account Detail Views**: Comprehensive transaction drill-down for each GL account with date range filtering, transaction type filtering, reference search, sortable columns (date, entry #, type, debit, credit), running balance calculation (Asset/Expense vs Liability/Equity/Revenue handling), drill-down links to source transactions (Receiving, Work Orders, AP, Invoices), and CSV export functionality. **Manufacturing Cost Flow**: Automated GL entries for Material Receiving (DR: Inventory, CR: A/P), Work Order Completion (DR: Finished Goods, CR: WIP), and A/P Payments (DR: A/P, CR: Cash). Access is role-based, primarily for Admin and Accountant roles.
