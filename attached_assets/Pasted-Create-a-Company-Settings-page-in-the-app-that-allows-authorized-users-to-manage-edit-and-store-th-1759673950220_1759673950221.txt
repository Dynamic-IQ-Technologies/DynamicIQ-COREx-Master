Create a Company Settings page in the app that allows authorized users to manage, edit, and store the organization’s general information. This data will be automatically used to populate the header section of generated business documents such as Purchase Orders, Sales Orders, Quotes, Work Orders, and Invoices.

Functional Requirements:

Company Information Management

Create a single Company Settings record (singleton record design).

Include editable fields for:

Company Name

DBA (if applicable)

Address Line 1

Address Line 2

City

State / Province

Postal Code

Country

Phone Number

Email

Website

Tax ID / VAT Number

DUNS Number (optional)

Cage Code (for aviation/government contractors)

Include Company Logo upload (image field) for display on document headers.

Add an “Updated By / Last Updated” tracking field for audit history.

Document Header Integration

All generated documents (e.g., Purchase Orders, Quotes, Invoices, Work Orders) should auto-populate header details from this Company Settings record:

Logo

Company Name

Address

Contact Information (phone, email, website)

Allow optional override per document (e.g., alternate business unit or division info).

Permissions & Security

Restrict editing access to users with Admin or Settings Management roles.

Allow read-only access to all users so that company information is viewable but not editable.

UI/UX Requirements

Page title: Company Settings

Use clean form layout with labeled sections:

“General Information”

“Contact Information”

“Tax & Regulatory Details”

“Branding” (for logo)

Include Save and Cancel buttons.

Display confirmation message after save:
“✅ Company information successfully updated.”

Optionally add a preview card showing how the header will appear on documents.

Automation & Document Binding

When generating documents (PDF or printable view), automatically pull data from the latest saved Company Settings record.

Ensure this data is consistent across all modules (Sales, Purchasing, Work Orders, etc.).

Automatically update existing templates to use these fields:

{{CompanyName}}

{{CompanyAddress}}

{{CompanyPhone}}

{{CompanyEmail}}

{{CompanyLogo}}
