Update the app to automatically generate and store an audit trail for all records created, updated, or deleted. The audit trail should be viewable as a secondary sub-tab under each record’s detail page.

Requirements & Functionality:

Automatic Audit Tracking:

Every time a record is created, edited, or deleted, the system should automatically create a corresponding Audit Trail Entry.

Captured fields should include:

Record ID

Record Type

Action Type (Created / Updated / Deleted)

Modified By (User)

Date & Time (System Timestamp)

Changed Fields (show previous and new values)

IP Address (optional for security audit)

Audit Trail Object / Table:

Create a new object or table named AuditTrail (or equivalent).

Link each entry to the corresponding parent record using a Record_ID lookup.

Ensure the table supports versioning and chronological sorting.

UI Implementation:

On each record page, add a secondary sub-tab labeled “Audit Trail” or “Change History.”

Within this sub-tab, display a read-only data table showing:

Action Type

Changed By

Date/Time

Changed Fields Summary

Include filtering by date, user, or action type for easier review.

Include a “View Details” option to expand and see the field-level change history.

System Logic / Triggers:

Implement backend logic (trigger, workflow, or event listener) that automatically writes to the AuditTrail table every time a record event occurs.

Ensure logic supports all modules in the app (e.g., Purchase Orders, Work Orders, Parts, Customers, etc.).

Avoid circular updates to prevent infinite audit loops.

Security & Access Control:

Restrict access to view the Audit Trail tab based on role permissions (e.g., Admins, Managers).

Ensure the audit data is immutable—no user should be able to modify or delete audit entries manually.

Performance & Maintenance:

Optimize database indexing for faster query retrieval.

Implement a data retention policy (e.g., retain audit entries for 7 years).

Enable export to CSV/PDF for compliance reviews.

Expected Outcome:

Each record automatically generates a detailed audit trail entry whenever any modification occurs.

Users can easily view a chronological, field-level change history under the “Audit Trail” sub-tab.

The feature improves transparency, accountability, and compliance tracking across all modules.
