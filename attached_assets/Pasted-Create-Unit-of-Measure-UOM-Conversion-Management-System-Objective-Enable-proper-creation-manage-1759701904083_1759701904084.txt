Create Unit of Measure (UOM) Conversion Management System

Objective:
Enable proper creation, management, and selection of unit conversions in the app, ensuring accurate quantities and costs when creating Purchase Orders or issuing inventory. Prevent any variance in inventory quantities, costing, or financial postings due to unit conversions.

1. Unit of Measure (UOM) Master
Fields / Attributes

UOM ID (auto-generated)

UOM Name (e.g., Each, Box, Kilogram, Liter)

UOM Type (Weight, Volume, Count, Length, etc.)

Conversion Factor (to base UOM, numeric)

Base UOM (standard unit for item, linked to Part Master)

Rounding Precision (decimal places for quantity)

Status (Active, Inactive)

Created By / Date

Last Modified By / Date

Behavior

All UOMs must link to a base unit for each part.

Conversion factors are immutable after posting to prevent accounting discrepancies.

2. Integration with Part Master

Each part has a Base UOM defined in the Part Master.

Allow multiple alternate UOMs with valid conversion factors.

Validate that conversion factor × alternate quantity = base quantity.

3. Purchase Order Integration

When creating a PO:

User can select any valid UOM associated with the part.

System calculates base quantity for inventory and financial posting:

Base Quantity = Ordered Quantity × Conversion Factor

Auto-calculate unit cost in base UOM:

Base Unit Cost = PO Unit Price ÷ Conversion Factor

Ensure total cost matches financial expectations.

Prevent variances:

If a PO is created in an alternate UOM, inventory quantity and cost must match base UOM calculations.

4. Inventory & Financial Integration

When receiving items:

Convert received quantity into base UOM.

Post correct inventory quantity and cost to GL.

When issuing items:

Convert issued quantity from selected UOM to base UOM.

Maintain accuracy in Work Order labor/material cost and COGS.

Rounding must be consistent to avoid discrepancies in financial postings.

5. Validations

Prevent deletion of UOMs used in existing transactions.

Prevent negative or zero conversion factors.

Validate numeric consistency:

Alternate Quantity × Conversion Factor = Base Quantity

Base Cost ÷ Conversion Factor = Alternate Unit Cost

6. UI/UX Requirements

UOM Master Page:

Create, Edit, View, Deactivate UOMs

Assign conversion factor and base UOM

Purchase Order Page:

Dropdown to select UOM

Automatic recalculation of quantities and costs

Inventory Page:

Display quantities in base UOM and optionally in selected UOM

Visual warning if UOM conversion would cause rounding or posting issues

7. Audit & Reporting

Maintain audit trail for all UOM creation, edits, and usage.

Standard reports:

UOM Usage Report (per part, per PO, per inventory transaction)

Conversion Factor Changes

Quantity and Cost Validation Logs

8. Expected Outcome

After implementation, your app will:

Allow proper creation and management of units of measure.

Enable selection of alternate UOMs in Purchase Orders, Inventory, and Work Orders.

Ensure no variances in quantities or financial transactions due to unit conversions.

Maintain audit-ready accuracy for inventory and accounting.
